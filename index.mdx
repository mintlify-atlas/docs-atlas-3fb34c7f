Taskiq is an asynchronous distributed task queue framework for Python. It takes inspiration from Celery and Dramatiq but is built from the ground up for async workloads — with full support for `asyncio`, FastAPI-style dependency injection, pluggable brokers, and a built-in cron scheduler.

## Why Taskiq?

Taskiq is designed for modern Python applications that rely on `asyncio`. Unlike Celery, Taskiq runs both sync and async functions natively, integrates seamlessly with FastAPI and other async frameworks, and exposes a fully typed API.

## Get started

<Steps>
  <Step title="Install Taskiq">
    Install the core library and choose a broker for your message queue backend.

    ```bash
    pip install taskiq
    ```
  </Step>
  <Step title="Define your broker and tasks">
    Create a broker instance and decorate functions with `@broker.task`.

    ```python broker.py
    from taskiq import InMemoryBroker

    broker = InMemoryBroker()

    @broker.task
    async def add(a: int, b: int) -> int:
        return a + b
    ```
  </Step>
  <Step title="Send tasks">
    Call `.kiq()` to send a task and `.wait_result()` to retrieve the result.

    ```python main.py
    import asyncio
    from broker import broker, add

    async def main():
        await broker.startup()
        task = await add.kiq(1, 2)
        result = await task.wait_result()
        print(result.return_value)  # 3
        await broker.shutdown()

    asyncio.run(main())
    ```
  </Step>
  <Step title="Start a worker">
    For distributed brokers, start a worker process to consume tasks.

    ```bash
    taskiq worker broker:broker
    ```
  </Step>
</Steps>
